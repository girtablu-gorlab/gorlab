# Dark Mode

Oddments uses a class-based dark mode. Adding `.dark` to `<html>` switches `color-scheme` from `light` to `dark`, which causes all of Skeleton's `light-dark()` paired tokens to resolve to their dark variant automatically.

## How It Works

```
<html data-theme="cerberus">          → light mode
<html data-theme="cerberus" class="dark">  → dark mode
```

**Step by step:**

1. `app.css` declares `@custom-variant dark (&:where(.dark, .dark *))` — the `.dark` class anywhere in the ancestor chain activates the `dark` variant.
2. Skeleton's `globals.css` sets `color-scheme: light` on `:root` and overrides it to `color-scheme: dark` under the `dark` variant.
3. All Skeleton paired tokens use CSS `light-dark(lightValue, darkValue)`. When `color-scheme` is `dark`, the dark value resolves.

## Paired Tokens

Skeleton generates paired tokens for every step in every palette — e.g. `--color-surface-50-950` resolves to `surface-50` in light and `surface-950` in dark. Use these in components instead of hardcoding a single step.

| Pattern | Light resolves to | Dark resolves to |
|---|---|---|
| `--color-surface-50-950` | `surface-50` | `surface-950` |
| `--color-surface-100-900` | `surface-100` | `surface-900` |
| `--color-surface-200-800` | `surface-200` | `surface-800` |
| `--color-surface-300-700` | `surface-300` | `surface-700` |
| `--color-primary-50-950` | `primary-50` | `primary-950` |
| `--color-primary-500-*` | `primary-500` | *— (use inverse pattern)* |
| `--color-primary-950-50` | `primary-950` | `primary-50` |

The naming convention is always `light-stop`–`dark-stop`.

## Writing Dark-Mode-Aware Components

Prefer paired tokens over hardcoded steps:

```css
/* ✅ Adapts automatically */
background: var(--color-surface-50-950);
color: var(--color-surface-950-50);

/* ❌ Only works in one mode */
background: var(--color-surface-50);
color: var(--color-surface-950);
```

In Tailwind classes, use the `dark:` variant modifier:

```svelte
<div class="bg-surface-50 dark:bg-surface-950 text-surface-950 dark:text-surface-50">
```
